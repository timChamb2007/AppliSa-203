Résumé

Résumé des Commandes git apprises :

  1. git add fichier/repertoire

    Cette commande permet de selectionne les fichiers pour ensuite faire un commit et les mettres dans le dépot distant.

  2. git commit -m "commentaire"

    Cette commande permet de prevenir le dépot distant que nous allons faire une modification tout en laissant un commentaire.

  3. git push

    Cette commande permet de transporter la modification du dépot local au dépot distant

  4. git status

    Cette commande permet de voir si un git add a déjà été effectué

  5. git log

    Cette commande permet de voir la liste des commits effectué depuis le début du dépot distant

  6. git pull

    Cette commande permet de transporter les modification effectué sur le depot github a l'intérieur du dépot local

  7. git diff

    Cette commande permet de voir les conflits du dépot git

  8. git clone

    Cette commande permet de cloner le depot distant a l'intérieur du repertoire ou l'on ce situe

  9. git init

    Cette commande permet de crée un dépot distant vide

  10. git stash

    Cette commande permet de faire des commits qui ne rentre pas dans l'historique (des commits aparts)

  11. git stash pop

    Cette commande permet de remettre les commits qu'on a fait a l'intérieur de la partie principale

  12. git stash list

    Cette commande permet de voir la liste des git stash qui ont été effectué et qui sont en attente

  13. git reset

    Cette commande permet d'annulez le dernier commit qui n'a pas encore été push

  14. git fetch

    Cette commande permet de recupéré le contenue d'une branche sans fusionner avec elle

  15. git bisect

    Cette commande permet de diviser les commits en deux afin de savoir qu'elle moitié fonctionne

  16. git cherry-pick

    cette commande permet de passer un numéro de commit et de le déplacer sur une autre branche

  17. git branch

    Liste toutes les branches locales

  18. git checkout NomDeBranche (déprécié)

    Sert à changer de branche

  19. git switch NomDeBranche

    Sert à changer de branche

Résumé des commandes docker apprises :

  1. docker ps

    Cette commande permet de lister les conteneurs en cours d'executions

  2. docker ps -a

    Cette commande permet d'afficher la Liste des conteneurs y compris les conteneurs arrêtés

  3. docker ps -q

    Cette commande permet de lister les id (le hashage) des conteneurs en cours d'executions

  4. docker run NomImage

    Cette commande permet de on crée un conteneur à partir d une image. Si nous n avons pas l image sur notre pc, elle est téléchargé automatiquement

  5. docker run -d -p Numport:80 NomConteneur

    Cette commande permet de lancer un conteneur en arrière plan accessible depuis le port Numport

  6. docker rm Numid

    Cette commande permet de le conteneur ayant l'id Numid (uniquement si il est arrêté)

  7. docker build -t NomImage

    Cette commande permet de créer une nouvelle image a partir du fichier Dockerfile

  8. docker run --name NomConteneur -d -p Numport:80 NomImage

    Cette commande permet de créer un conteneur accesible au Numport qui a comme image NomImage
